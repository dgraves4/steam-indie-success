Data Cleaning and Exploratory Data Analysis (EDA) for Steam Indie Games Dataset¶

1. Introduction¶

The objective of this notebook is to clean, preprocess, and perform an exploratory data analysis (EDA) on the dataset of indie games from the Steam platform. This dataset, collected through the Steam Web API, contains comprehensive metadata about various games, including features like price, genre, developer information, release date, and user engagement metrics.

By utilizing effective data cleaning and EDA techniques, we aim to understand the key characteristics of the dataset, detect patterns and correlations, and prepare a high-quality dataset for the predictive analysis phase of this project. The insights gained in this phase will help determine which features are most relevant for predicting game success and guide us in the development of machine learning models.

2. Objectives of the Notebook¶

  1. Data Cleaning:

    • Handle missing values and correct inconsistencies in the dataset.
    • Engineer relevant features to provide additional context and improve the predictive capabilities of the dataset.
    • Normalize features to ensure that the dataset is prepared for advanced analysis and machine learning.

  2. Exploratory Data Analysis (EDA):

    • Visualize the distribution of key numerical features to understand underlying patterns.
    • Identify relationships between features, such as price, user recommendations, and genre.
    • Use visual tools such as distribution plots, correlation heatmaps, and boxplots to explore the data and inform feature selection.

3. Data Cleaning Overview¶

The cleaning process involves addressing missing values, engineering new features, and scaling numerical variables. Key transformations include:

  • Handling Missing Values: Imputing missing numerical values and removing records that lack critical information.
  • Feature Engineering: Adding new attributes, such as "Years Since Release," to provide additional temporal context.
  • Normalization: Scaling key numerical features using Min-Max normalization to maintain consistency and aid in predictive modeling.

This section is critical in ensuring that the data is complete, accurate, and prepared for the next phase of analysis.

4. Exploratory Data Analysis Overview¶

In this section, we focus on exploring the dataset to gain deeper insights and identify potential relationships among features:

  • We employ distribution plots to visualize pricing trends, heatmaps to understand correlations among features, and boxplots to explore the variations in price across different genres.
  • These visualizations allow us to formulate hypotheses about what might influence game popularity and success, which will be valuable for model building in subsequent stages.

5. Notebook Structure¶

  1. Data Loading: Load the dataset and perform initial checks to understand its structure.
  2. Data Cleaning: Process the raw data to handle missing values, engineer features, and prepare it for modeling.
  3. Exploratory Data Analysis (EDA): Use various visualizations to analyze the features of the dataset, identify trends, and explore relationships.
  4. Conclusion: Summarize findings from the EDA and discuss their implications for the predictive analysis phase.

6. Next Steps¶

The insights gained from this notebook will serve as the foundation for the next stage of the project: Predictive Analysis. Using the cleaned and preprocessed dataset, we will develop machine learning models to predict the commercial success of indie games on Steam. The features identified in the EDA phase will play a key role in building effective models.

By the end of this notebook, we aim to have a clean, well-understood dataset that is ready for machine learning modeling, and a solid grasp of the data's underlying patterns and relationships.
